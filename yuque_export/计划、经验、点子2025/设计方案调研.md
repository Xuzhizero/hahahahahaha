# teamviewer支持在同一个wifi子网下远程控制被控端电脑。隔墙与不隔墙是否会显著对控制能力产生影响？比如被控电脑和主控电脑一台在楼上，一台在楼下，但是通过路由器组网被覆盖在同一个wifi子网下（楼上楼下信号都很良好），它们之间用teamviewer是否仍然可以实现良好的通信？
在同一 Wi‑Fi 子网（局域网）内使用 TeamViewer 时，两个端点之间的控制流量会优先走点对点（P2P）或经由本地路由器转发，而**不会**先上传到云端再回传，因此延迟和带宽完全取决于你家的无线网络质量，而不是墙壁“隔不隔”本身。

**1. 墙壁只是影响 Wi‑Fi 信号的物理因素**

+ 墙体（尤其是承重墙、混凝土墙）会对 5 GHz 频段的信号衰减更明显，但只要你在楼上楼下测得的 RSSI（信号强度）在 –60 dBm 以上，丢包率和重传都很低，用户几乎感知不到延迟或卡顿。
+ 如果信号都保持在良好水平（RSSI ≥ –60 dBm，相当于“满格”或“3 格”以上），隔墙与否对 TeamViewer 的控制性能并**不会**带来明显差异。

**2. 关键影响因素：网络质量，而非物理位置**

+ **吞吐量**：TeamViewer 传输屏幕更新和音视频需要数 Mbps 带宽。确保 AP 支持 802.11ac/ax 且网络空闲时带宽充足。
+ **延迟**：局域网内 P2P 连接通常只有 1–5 ms 的往返延迟。
+ **丢包率**：如果丢包率超过 1%，会引起画面卡顿或操作延迟重传。丢包通常由信号弱、干扰（其他 Wi‑Fi、微波炉、蓝牙）或 AP 负载大引起。

**3. 楼上楼下的最佳实践**

+ **选择合适频段**：
    - 2.4 GHz 穿透力强，覆盖范围大，但带宽较低，干扰多；
    - 5 GHz 带宽更高，干扰少，但穿透力弱。
    - 如果墙太厚或距离较远，可切到 2.4 GHz，确保信号稳定；日常靠近 AP 时优先 5 GHz。
+ **优化网络布局**：
    - 使用 Mesh 或无线中继器，保证各楼层信号“满格”；
    - 若条件允许，上下楼各放一个小型无线路由，组建同一 SSID（接入点漫游），把楼上楼下 AP 之间的有线回程接到主路由。
+ **开启局域网模式**
    - 在 TeamViewer “高级”设置里打开 “仅限局域网连接（Incoming LAN connections）”，可以强制使用 LAN，进一步减少经过外网的可能。

**结论**：只要你的路由器对上下两层都能提供“良好”甚至“满格”的 Wi‑Fi 信号，隔墙并不会显著影响 TeamViewer 的操控体验。测试时，不妨先在两台电脑上 ping 一下彼此，确认延迟在 5 ms 以内、丢包率接近 0，再开启 TeamViewer，即可获得接近有线局域网的顺畅远控效果。
