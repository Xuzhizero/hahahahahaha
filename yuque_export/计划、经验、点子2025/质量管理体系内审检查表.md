# 
**编制说明：** 本检查表旨在通过内审推动各部门建立和完善关键文档体系，提升管理规范性和可追溯性。

---

## 一、智能船体研究所 内审检查内容
### 1. 设计开发过程文档化（条款8.3）
#### 1.1 设计开发策划文档
- [ ] **<u>检查要点</u>**<u>：是否建立《设计开发计划书》模板？</u>
- [ ] **文档建议**：建立标准化的项目计划书模板，包含：
    - 项目阶段划分（概念设计、初步设计、详细设计、施工设计）
    - 各阶段时间节点和里程碑
    - 各阶段负责人和参与人员
    - 评审、验证、确认节点安排
- [ ] **审核方法**：抽查2-3个在研项目的计划书

#### 1.2 设计输入管理
- [ ] **检查要点**：<u>是否有《设计输入评审表》？如何确保输入的完整性？</u>
- [ ] **文档建议**：建立设计输入清单，包含：
    - 客户技术规格书接收记录
    - 法规标准要求识别清单（CCS规范等）
    - 类似项目经验总结文档
    - 设计输入评审会议纪要
- [ ] **审核方法**：查看最近项目的输入文档完整性

#### 1.3 设计评审记录
- [ ] **检查要点**：各阶段设计评审是否有规范的记录表格？
- [ ] **文档建议**：制定《设计评审记录表》，包含：
    - 评审阶段和日期
    - 参会人员签到表
    - 评审发现的问题清单
    - 问题整改跟踪表
    - 评审结论和批准记录
- [ ] **审核方法**：抽查概念设计、详细设计阶段的评审记录

#### 1.4 设计变更控制
- [ ] **检查要点**：设计变更是否有完整的审批流程记录？
- [ ] **文档建议**：建立《设计变更申请单》和《变更影响分析表》：
    - 变更原因说明
    - 变更内容详述
    - 影响范围分析（成本、进度、其他系统）
    - 相关方确认签字
    - 变更实施记录
- [ ] **审核方法**：追溯最近3个月的设计变更记录

#### 1.5 图纸和技术文档管理
- [ ] **检查要点**：<u>图纸送审记录是否完整？版本控制是否规范？</u>
- [ ] **文档建议**：建立《图纸送审记录表》和《图纸发放登记表》：
    - 送审图纸清单
    - CCS审图意见和回复
    - 图纸版本更新记录
    - 图纸发放和回收记录
- [ ] **审核方法**：检查图纸管理台账的完整性

### 2. 外包过程控制（条款8.4）
#### 2.1 外包项目管理
- [ ] **检查要点**：<u>外壳表面处理、船舶动力仿真等外包项目是否有质量控制记录？</u>
- [ ] **文档建议**：建立《外包项目质量控制表》：
    - 外包方资质审查记录
    - 技术要求交底记录
    - 过程检查记录
    - 验收标准和验收记录
- [ ] **审核方法**：查看近期外包项目的全过程记录

### 3. 技术服务和交付后活动（条款8.5.5）
#### 3.1 技术服务记录
- [ ] **检查要点**：现场技术服务是否有完整的记录？
- [ ] **文档建议**：完善《施工流程单》，增加：
    - 服务申请和派工记录
    - 现场问题诊断记录
    - 技术服务报告
    - 客户确认签字
    - 后续跟踪记录
- [ ] **审核方法**：抽查3份技术服务记录的完整性

#### 3.2 项目交付文档
- [ ] **检查要点**：<u>项目交付时的技术文档是否齐全</u>？
- [ ] **文档建议**：建立《项目交付文档清单》：
    - 设计图纸（含审批记录）
    - 计算书
    - 技术规格书
    - 操作维护手册
    - 培训记录
- [ ] **审核方法**：检查最近完工项目的交付文档

### 4. 产品标识和可追溯性（条款8.5.2）
#### 4.1 项目文档编码体系
- [ ] **检查要点**：<u>是否建立统一的文档编码规则？</u>
- [ ] **文档建议**：制定《文档编码管理规定》：
    - 项目编号规则
    - 图纸编号规则
    - 技术文档编号规则
    - 版本管理规则
- [ ] **审核方法**：检查编码规则的执行情况

## 内审实施建议
### 1. 内审前准备
+ 提前一周通知被审核部门，让其准备相关文档
+ 提供文档模板样例，便于部门参考建立
+ 明确本次内审的重点是"帮助建立文档体系"而非"挑毛病"

### 2. 内审中引导
+ 发现文档缺失时，现场讨论制定文档框架
+ 对已有但不规范的文档，共同探讨改进方案
+ 记录部门的实际困难，协助寻找解决方案

### 3. 内审后跟进
+ 形成《文档体系建设计划》，明确时间节点
+ 建立文档模板库，供各部门共享使用
+ 定期回访，协助解决建设过程中的问题
+ 对完成情况好的部门给予表彰

### 4. 重点推进事项
**智能船体研究所**：

+ 优先建立设计评审和变更控制体系
+ 规范图纸管理流程

**智能机舱研究所**：

+ 优先建立软件配置管理体系
+ 完善系统测试和验证文档

**综合办公室**：

+ 优先完善供应商管理体系
+ 规范培训效果评估机制

5. 为后续外审打下良好基础
