# 试验方面总结
1. **雷达系统性能**
    - 辽无二 LR19 系列 X 波段雷达在真实海况下（2–6 级）对多类典型目标均具备稳定探测与跟踪能力，包括大型船舶、近距离船只、小型渔船、岛屿及岸线。
    - 推荐参数设置：
        * **开阔海域 5–6 级**：量程 3 海里 / 增益 ~65% / 海浪抑制 ~75% / 雨雪抑制 ~30%。
        * **近岸 2–4 级**：量程 ≤1.5 海里 / 增益 ~60% / 海浪抑制 ~65% / 雨雪抑制 ~30%。
    - 存在两点局限：①浪涌时岸线连续成像偶有分裂；②大幅快速转向时偶发目标跟踪丢失。
2. **循迹与自动舵试验**
    - **8月1日手动循迹（800转）**：在多组不同航向偏差与风浪条件下，绝大部分试验航迹跟踪效果良好
    - **自动舵算法**：航向保持稳定，控制效果良好。
    - **8月2日自动/手动切换与不同转速试验**：650 转速下循迹基本可行但效果一般；800 转速存在油门过冲与操舵异常。
    - 多次切换自动与手动模式时，若出现操舵问题，后续试验均无法正常循迹。
3. **GPS信号全程正常，未发现异常现象**
4. **异常问题与处置**
    - 工控机掉电无法启动（UPS 掉电）→ 手动重启解决。
    - 转速反馈线未接→ 重新接线后恢复。
    - 档位操纵杆偶尔无法挂挡→ 调整零位解决。
    - 油门操纵杆拉升不精确→ 造成转速过冲（未根本解决）。
    - 手动车钟 800 转速下操舵失灵→ 未能解决。
    - 雷达偶尔丢失罗经信号→ 软件调整后解决。
    - 远海区域（距离东福山20km）曾失去 Wi-Fi/4G 信号→ 返航至信号恢复区。



# 船长反馈情况
1. 目前的辽无2雷达虽然性能更好，但是操作过于复杂，大多数操作需要用到鼠标，船长更喜欢传统雷达旋钮和按键式的雷达
2. 电子海图界面的操作也是类似，可以操作，但是由于需要用到鼠标，船长用不习惯
3. 建议只开放规划航线和操舵功能即可，排斥自动车钟功能



# 后续实验建议（技术人员重点关注）
1. 需要两个人，一个人操作，另一个人速记，否则容易手忙脚乱，导致对于一些情况，容易漏过。两个人最好都了解情况，而不是让一个不了解情况的人记录数据。
2. 海试报告最好先经由海试出发前的各（负责）人先经过确认，确保消除歧义，然后再提交终稿
3. Ubuntu 默认 Ctrl + Alt + Shift + R  录制的视频格式文件是WEBM格式，相比起MP4，**默认关键帧间隔比较长**（往往要5 秒甚至更长），所以快进时需要解码很多帧 → 反应慢，不方便做后续分析。  以后录制视频需要事先将默认录制格式改为MP4。具体方法可直接问AI
4. 如果录制的是webm格式文件，建议用浏览器打开。 因为WebM 本身就是为网页视频而设计的  。
