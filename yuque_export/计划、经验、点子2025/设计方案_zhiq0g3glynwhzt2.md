# 概述
为了大幅度减少实船测试频率和调试用时，极大加快智能航行算法的预研效率，仿真对于智能船舶研究院的未来发展极为关键。目前智能航行所已经初步搭建了仿真环境，可以初步满足仿真需求，但是各个功能模块都运行在自己的工作主机中，这样带来的第一个问题是，硬件上和最终实船上搭载的硬件设备和网络结构都有很大的区别，缺乏系统级的验证。如果能够实现对标实际船只的接口和网络链路，可以大幅度缩短实船上的调试时间。

因此，有必要在船舶研究院设立一个专门的仿真实验室，对标实际船只的网络和软硬件连接架构，容纳和链接智能航行所的各个功能组件。建立仿真实验室，还有下面的两大意义：

1. 对于智能机舱的功能调试（例如对主机和电力系统的调试），也有很大的在本地实现测试的必要，需要专门的空间容纳相关的机电硬件设备和仿真设备，更重要的是，通过仿真实验室，可以实现智能机舱和智能航行的系统集成，实现更系统性的整体调试
2. 仿真实验室是绝佳的对外展示平台，对于外宾和领导展示，会展展示都具有重要意义。

综上，仿真实验室建设具有重要意义。后文将具体阐述仿真实验室的建设规划。

# 仿真实验室中仿真机概述
仿真实验室最核心的部分，当然是仿真计算机。仿真计算机的作用主要体现在两个方面：

1. 通过船体和环境的模型模拟实际世界的运行规律
2. 通过软件程序或者信号发生器提供模拟的感知数据

具体的建模精度和提供的感知数据的类型和丰富程度，都取决和服务于我们的算法开发和调试验证需求。现将开发需求罗列如下：

| 开发组别 | 验证算法 | 最理想情况下的仿真功能需求 |
| --- | --- | --- |
| 基本需求 |  | 1. 仿真中的物体尺度、空间尺度与实际一致   2. 支持直接导入实船 CAD 图并据此直接得到关键模型参数   3. 能在仿真中添加标识：期望点、期望路径、实际路径   4. 导入真实海洋场景（如舟山海域） |
| 规划与控制 | 循迹算法 | 模拟不同等级、方向的风浪扰动 |
| 规划与控制 | 极端工况下运动控制算法 | 1. 满舵后反向满舵测试（紧急避碰、恶劣海况）   2. 不同风浪等级与方向下船体六自由度抖动/摆动模拟 |
| 规划与控制 | 避碰算法 | 1. 添加随机或预定路径移动的障碍船只/跟踪目标   2. 添加海面与海底障碍（浮筒、岛礁、灯塔），电子围栏具备海底坡度 |
| 规划与控制 | 自主靠泊算法 | 1. 支持多种地形环境：港口、狭窄水域等   2. 不同等级、方向风浪条件下靠泊动作 |
| 感知定位 | 图像视觉增强算法 | 1. 不同光照条件下视觉效果   2. 不同天气条件下视觉效果 |
| 感知定位 | 雷达–视觉融合算法 | 雷达回波仿真及接收    |


上表将作为后续选择仿真技术方案和开发顺序的重要抓手。在仿真技术架构的选择上，从定性上说，从构建伊始就应该要立足下面因素：

1. 全面性。动力学仿真模型框架要在搭建伊始就具备涵盖几乎所有海况的理论技术条件，且要覆盖船舶运动的六个自由度
2. 仿真数据易得。仿真模型中的数据方便获得，方便二次处理，方便可视化
3. 传感器高精度。模型中需要加入物理特性高保真的传感器
4. 易扩展。如果未来有新的传感器添加需求，或者有新的动力学模型改进需求（例如将舵-桨驱动方式改成直翼舵桨驱动方式，单个直翼舵桨驱动改成多个直翼舵桨驱动），在模型的更改上需要足够方便
5. 模型递推快速性。由于未来需要纳入多种复杂的海况，因此模型的复杂度在未来一定会很高，因此模型递推的解算速度需要足够快
6. 高保真可视化。为了给感知算法提供高保真的感知背景，环境可视化需要具有较高的保真度
7. 实时性。搭建在不同主机之间的数据沟通需要确保低时延，例如搭载有算法的主机和专门的仿真机之间的数据互通
8. 迭代快速。整个项目工程在缺少人力的前提下，需要能够快速迭代。这一点可以依靠成熟的开源代码、成熟的社区生态等因素支撑
9. 易协同。算法测试人员如果需要仿真，可以方便进行协同仿真，他们为了仿真而额外学习的与仿真部分的内容需要足够少，协同仿真尽可能方便
10. 低成本。采用的方案对经费的开销需要足够小。

仿真技术架构一旦确定，将在很长时间沿用这个架构，否则将会造成大量的时间精力浪费。经过充分调研，拟采取Simulink+UE5的仿真技术架构，即模型基于Simulink 全权计算，包括风浪的数据也是在simulink里实现，然后交由UE5渲染可视化，UE5也可以反过来为Simulink提供视觉和雷达的感知数据。下面从前述十个定性指标对此技术架构进行评估：

| 维度 | 方案：纯 Simulink 驱动 → UE5 接收 |
| --- | --- |
| **全面性** | 高，配合开源的“海洋系统模拟器”Simulink模块，可在 Simulink 中搭建包含各类海况的理论模型，支持完整水动力理论、基于CFD 标定数据。 |
| **仿真数据易得性** | 白盒，所有数据可以获得，且易通过Matlab进行数据处理/解析和可视化 |
| **传感器高精度** | 可引入高保真物理传感器模型（LIDAR、水速计和摄像头等） |
| **易扩展** | 高，模块化清晰，新增传感器或动力学模型相对方便，**物理模型与可视化彻底解耦。** |
| **模型递推快速性** | 中等，推算速度受模型复杂度影响，但是可以通过更高性能的计算机克服 |
| **高保真可视化** | 良好，只做数据驱动，波浪仅作视觉效果，船体不随波动真实晃动 |
| **实时性** | Simulink→UE5在同一个操作系统，时延低 |
| **迭代快速** | 大量现成块和教程，生态丰富，社区成熟，迭代效率高 |
| **易协同** | 中等，需具备 Simulink和MSS等海洋学相关库 操作能力 |
| **低成本** | 较低， UE5 免费， MATLAB/Simulink 许可可以通过渠道低成本获取 |


综上，由于我们对模型可调节性和对环境数据的获取（用于算法定量分析）提出了一定要求，同时对迭代效率和社区支持有较强需求，综合考虑人力与时效，可采取此架构，以保证高精度、可深度调试与扩展的动力学与传感器模型，期间可以通过简单的 UE5 波浪插件补充视觉效果。它的缺点是，Simulink与UE5画面交互匹配度无法保证（UE5中有浪，但是画面上没有相应的船体波动效果），但是后续或许可以考虑通过水体粒子采样的方法解决这个问题。

由于仿真中模拟的物理环境、被控对象的物理特性和实际必定有所差别，总的来说，仿真内容的开发投入和回报大致满足下面的关系：



![](https://cdn.nlark.com/yuque/0/2025/png/2408029/1757659882337-d4ead990-9e95-4937-b145-62999179587b.png)

可以看出，仿真开发时间会随着仿真的拟真度的增加而陡然上升，然而作为回报，后期实船调参耗时却下降地愈发平缓。鉴于暂时可供调用的资源相当有限，考虑"逐步递进，分期开发"的开发路线，即优先解决最紧迫且相对容易解决的仿真需求，再逐步过渡到更为复杂的仿真任务上。

# 第二阶段仿真实验室建设规划
依靠第一阶段奠定的基础，相比第一阶段，网络架构已经不需要做太多调整，主要是对仿真机的仿真功能进行优化。在这一阶段，仿真机的改进可以分为模型和感知数据两个方面。

1. 模型。引入六自由度动力学模型，引入执行机构（螺旋桨）推力曲线模型，引入流速流向模型；引入负载放置位置对船体参数的影响
2. 感知数据。引入感知数据来源（UE5）并支持可视化展示；在UE5中引入目标/障碍物位姿和轨迹生成功能；引入不同光照与天气场景；引入雷达回波仿真功能并能可视化展示；引入水深数据。感知模块可以接收这些数据。

对应的算法开发需求列表如下：

| 待开发的智能航行功能 | 验证算法 | 对应仿真功能需求 |
| :---: | :---: | --- |
| 规划与控制 — 自主循迹 | 循迹算法 | 在常规水流条件下沿预设航线进行路径跟踪 |
| | | 可视化期望点/期望路径与实际轨迹 |
| 规划与控制 — 狭窄水域定向定点 | 路径规划算法 | 左右岸线与狭窄水域边界建模 |
| 规划与控制 — 动态避障 | 避障算法 | UE5中添加静态障碍物（包括桥、浮标、岛礁）、动态移动的障碍船只（游艇、渔船）。 |
| 感知定位 — 图像目标跟踪 | 图像识别算法，视觉增强算法，视觉和雷达数据融合算法 | 模拟多种光照（晴/背光/夜晚）和天气（雾/雨/阴天）条件，且光照和天气随时间和季节、大致经纬度对应变化；模拟雷达回波 |
