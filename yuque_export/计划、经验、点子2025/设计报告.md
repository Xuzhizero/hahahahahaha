# 概述
## 立项背景
在舟山市政府的委托下，浙江省智能船舶研究院有限公司（以下简称“研究院”）基于现有的“适应高海况长航程海上大型无人艇自主航行关键技术研究”成果，承担了“大型无人艇概念设计、仿真系统和中试基地建设”这一任务。该项目旨在进一步推动无人艇的概念验证与实际应用能力，并为未来的海上智能船舶研发和测试提供坚实的基础设施支持。

## 项目目标  
本项目的核心目标是设计并验证一种适合复杂海况和长航程任务的大型无人艇概念。通过构建仿真系统，我们希望在虚拟环境中对无人艇的设计方案进行充分的验证和优化。此外，通过建设中试基地，为后续的实船测试和性能评估提供一个可靠的实验平台。

## 设计原则
在设计过程中，我们将遵循模块化、可扩展性以及安全可靠性原则，确保无人艇设计既能适应多变的海洋环境，也能方便后续的维护和升级，同时我们将重点突出人工智能技术在概念设计和仿真过程中的赋能作用，以提升整个系统的智能化水平和自动化能力。

## 预期成果
预期通过本项目，将形成一套完整的大型无人艇概念设计方案和仿真验证体系，并建成一个可用于中试测试的实验基地。最终的成果将为相关政府部门和业界提供一个可行的参考范例，并推动无人艇技术的实际应用和产业化进程。

# 总体设计
本系统围绕大型无人艇在多样化海洋任务中的智能执行能力展开设计，采用“船岸协同、虚实融合、任务驱动”的总体思路。系统架构划分为三层：基础支撑层、交互协同层和任务能力层，整体关系如图所示：

### 一、基础支撑层：岸端模型与船端模型
系统的底层由“岸端模型”与“船端模型”共同构成，分别承担计算与控制、数据处理、感知决策等核心职能：

+ 岸端模型主要包括数据库、算力中心和仿真环境三大模块，提供算法训练、数据回放、任务模拟等能力，是系统的“计算大脑”和“仿真平台”。
+ 船端模型则部署在实际的无人艇平台上，以真实的传感器、控制器和执行机构为依托，面向实海场景执行任务。
+ 同时，测试场作为一个标准化、开放性的场地环境，为无人艇的部署、调试与实地验证提供现实支撑。

### 二、交互协同层：三类交互形式
连接岸端与船端的是三类交互机制，它们共同构成支撑上层任务实现的“协同通道”：

+ **数字孪生**：通过仿真数据与实船数据的双向同步，实现“虚实映射”，既可提前模拟评估算法效果，也可回放分析实海数据，增强系统的验证能力；
+ **远程通信**：保障船岸之间的命令下发与状态上传，支持远程监控、远程控制等关键功能；
+ **端边协同决策**：岸端系统能够通过端边协同的机制和高算力优势，准确识别无人艇所处的具体环境模式，并相应的为船端更新一套适合当前情境的导航和控制参数，以确保无人艇能灵活应对不同的任务场景。

三类交互形式彼此关联，相互协作，构成了系统运行过程中的核心数据流与控制流。

### 三、任务能力层：四大任务支撑
在上述系统架构的基础上，平台支撑四大典型任务能力的实现：

+ **远程操控**：操作人员通过岸端系统实现对无人艇的远程驾驶与状态感知；
+ **自主循迹**：船端系统基于路径规划与控制算法，自主完成固定航线的精准跟踪；
+ **自主避障**：集成目标识别与局部路径重规划算法，提升在复杂环境下的自主航行安全性；
+ **集群协同**：多艇之间通过感知共享、意图协商等机制，实现分布式协同作业。

四项任务各自依赖于不同程度的船岸资源调度和交互机制支撑，是系统智能水平的集中体现。

# 船端自主航行系统介绍
在整个系统的总体架构中，船端模型是整个体系的终端载体，它是最终实际执行自主航行任务的部分。也就是说，所有规划和决策最终都会在这个船端模型上得到实际的执行和验证。而在船端系统内部，自主航行系统则是船端模型的核心组成之一。它负责将岸端下达的任务指令转化为具体的航行动作，通过环境感知、路径规划和控制执行这三大模块来确保无人艇能够自主、安全地完成预定的航行任务。下面我们将分别对这三个模块进行具体介绍。



## 环境感知模块
环境感知模块的功能是实现对平台周围环境的实时感知，包括目标识别、多源信息融合和环境建模。具体功能如下：

（1）目标识别：通过YOLO算法基于视觉识别轮船、客船、渔船、快艇、浮标、码头、助航设施等目标。

（2）多源信息融合：通过X波段雷达与光电摄像头、毫米波雷达与光电摄像头的数据融合，提高感知系统的准确性和可靠性，为自主航行系统提供路径规划和避碰决策支持，实时检测平台周围水域的静止和动态障碍物。

（3）环境建模：船舶自主靠离泊需精确感知泊位、岸壁及障碍物的三维空间关系，通过激光雷达进行SLAM建模，同时，可以获取靠离泊过程中需要的泊位信息、其他船舶位置信息、周边物体信息。

### 硬件组成
感知分系统由以下主要部分组成：

（1）X波段雷达：用于远距离目标检测和跟踪。

（2）毫米波雷达：用于近距离目标检测和跟踪，特别是在狭窄水域操作中。

（3）光电跟踪定位系统：包括红外摄像头和可见光摄像头，用于目标的精确识别和跟踪。

（4）激光雷达：用于高精度的环境建模和障碍物检测。

（5）AIS系统：用于接收其他船舶的自动识别信息。

（6）摄像头：与毫米波搭配使用，通过互补性弥补单一传感器的性能短板。

（7）360全景影像：提高作业人员对船舶周边近距离环境的感知与判断。

（8）融合处理计算机：对系统中的多源感知信息进行融合处理、存储、发送。

（9）闭路电视：对船舶机舱内、前后甲板实时监控。

**结论**

感知分系统通过多种传感器的融合，实现了对平台周围环境的实时感知和目标识别，为自主航行系统提供了准确的环境信息。通过X波段雷达和光电、毫米波雷达和光电的结合、激光雷达SLAM建模，系统能够在远距离和近距离场景下都能有效工作。在狭窄水域操作中，毫米波雷达提供了高精度的近距离感知能力。在靠离泊中，激光雷达SLAM进行准确环境建模，提供高精度的位置和地图信息，确保船舶能够安全、精准地完成靠泊和离泊操作。<font style="color:rgb(238,0,0);"></font>

## 规划模块
规划模块主要负责路径的制定和优化，旨在为船舶提供一套智能化、自动化的航线生成解决方案，主要涵盖自动航路规划这一核心功能。目前我们在规划模块中搭载了两种主要的规划算法：

+ A 算法*：这是一种全局路径规划算法，能够根据已知的全局环境信息，为无人艇规划出一条从起点到终点的最优路径。
+ 人工势场法（APF）：这是一种局部路径规划方法，特别适用于动态避障和局部环境变化的场景，通过模拟吸引力和排斥力来引导无人艇避开障碍。

这两种算法的组合使得无人艇既能在全局上有清晰的路径，又能在局部环境中灵活应对障碍物。



自动航路规划的核心在于，用户通过交互界面输入航行的起始点与目标点经纬度、本船的实际吃水深度，并可选择性地设定如是否规避他国领海等特殊航行限制。此外，系统还允许用户根据临时航行通告或特定需求，在电子海图上手动绘制并添加额外的不可航行区域。根据用户需求，系统可以生成最短距离路径以及最佳绕岛路径。当船舶在根据选定航线行驶的过程中，如果偏离航线最大偏航距离，系统需要实时调整航线，并且实时分析碰撞/搁浅危险。



系统接收到任务规划请求后，会首先基于其内部处理的网格化海图数据进行分析。此网格化地图已预先提取了每个网格单元的关键信息，包括水深、陆地标识、已知碍航物以及是否属于特定管制水域（如他国领海）等。系统依据用户输入的船舶吃水、选择的避让规则以及预设或用户自定义的不可航区域（包括水深不足区域、陆地区域、默认及用户指定的危险物标图层覆盖区、他国领海区域、手动添加的禁航区等），动态确定当前任务下的可航行与不可航行空间。







## 控制模块
控制模块负责将规划好的路径转化为无人艇的实际动作。目前我们在控制模块中主要采用了 PID 控制方法。PID 控制器通过调节比例、积分和微分参数，实现对无人艇航向、速度等的平稳控制，确保无人艇按照规划好的轨迹稳定航行。



# 岸端系统介绍
在整个系统的总体架构中，岸端系统扮演着重要的后端支撑角色。它主要由仿真环境、算力中心和数据库三大部分组成，因此它的核心作用在于：第一，提供仿真支持，能够模拟各种海上场景和任务；第二，提供算力支持，为算法运行和模型训练提供强大的计算能力；第三，负责数据存储和管理，确保所有实验数据和历史记录的安全和可追溯。岸端系统作为后台中枢，正是通过这三个关键模块来支撑和保障船端任务的顺利进行。下面具体介绍这三个组成部分的功能和特点。

## 仿真分系统
系统总体可分为模型递推模块和可视化模块，仿真流程图如下图所示：

![仿真流程图](https://cdn.nlark.com/yuque/0/2025/png/2408029/1753324938961-9206d663-6b88-4c7c-856f-c21c8ec29a8c.png)

具体解释如下：

在同一个WIFI子网下，ROS通过 ROS Topic 发出推力/力矩指令给 Simulink，Simulink 递推得到速度和位姿后再同时发回 ROS（给算法验证）和 UE5（更新可视化场景），而 UE5 又能把局部波浪采样结果返传给 Simulink，形成宏观与微观环境的双向耦合。



在 Simulink 中，每个仿真步首先读取来自 ROS 的控制命令，然后将推进力、环境干扰（包括 UE5 返回的粒子采样风速／波速信息）一并带入六自由度动力学模型，求解线速度和角速度并积分得到位姿。Simulink 以高频率跑通这套动力学＋运动学，通过ROS Topic把最新姿态推送给上层算法，也推给 UE5 用于渲染同步。



UE5 方面接收到 Simulink 的位姿和Simulink预先设定的风速、显著波高、波谱类型、地形高度图等宏观参数后，会实时把船体放到三维地形上，利用海面流体材质和粒子特效做到从大尺度波浪到船体周边溅水的高保真渲染，并采集船体周边的局部风浪信息（可能区别于总体的风浪布局）；同时，UE5内置流体插件还可以简易地推演其他“来船”目标的位置轨迹，丰富场景动态。UE5 的传感器插件（激光雷达、摄像头、IMU 等）则在完整场景里读取几何、光照、天气、动态目标等信息，生成点云、图像、惯性测量等原始数据，再返回给 Simulink／ROS，为感知与定位算法提供闭环验证。



这样一来，我们就实现了“ROS→Simulink→UE5→ROS”的端到端仿真链路：算法在智能航行分系统（包含两台Ubuntu主机）上决策，模型递推分系统（以Simulink为主体） 在 Windows 上做高频物理推演，可视化分系统（以UE5为主体） 在同一台机器上做高保真渲染与传感器仿真，并通过粒子采样将局部环境细节回馈给动力学模块。宏观参数驱动大尺度海况，粒子群采样补足微观波浪，最终在多频率、多主机、多模型之间形成一个动态一致、可视可测的自主航行仿真闭环。

### 模型递推模块
搭载于Win11主机，基于MATLAB&Simulink的 模型分析系统是整个仿真平台的“推递仿真”核心，承担以下主要功能：

1. **推进力和力矩计算**  
在“执行机构模型”中接收来自 ROS 的推进指令（如螺旋桨转速、矢量推进器方向），并转化为对应的推进力和力矩，为动力学计算提供输入。
2. **环境干扰计算**  
通过“环境干扰模型”结合“解算器”，使用风、波宏观参数（及可选的 UE5 局部采样信息）精确地计算附加在船体上的水动力干扰力矩，涵盖波浪冲击、洋流作用等多种海况效应。
3. **船体动力学仿真**  
在“船体模型”中建立六自由度动力学方程，耦合推进与干扰力后求解线速度和角速度，并在“运动学模型”中将速度积分得到实时位姿，完整再现船体在复杂海况下的运动行为。
4. **状态输出**  
将最新的位姿信息实时发送给 UE5，取得高保真的可视化效果；同时将船体真实状态反馈给 ROS，用于算法性能评估和闭环验证。
5. **传感器虚拟化**  
借助“各类传感器模型”，生成仿真激光雷达点云、摄像头图像、IMU 数据等多源传感器输出，为感知与融合算法提供真实感知输入。
6. **统一仿真步长**  
通过锁定仿真步长，保证与 ROS 算法模块和 UE5 可视化模块之间高一致性的数据流通。

### 可视化模块
搭载于Win11主机，基于UE5的可视化分系统是整个仿真平台的“感知与展示”核心，它不仅为物理仿真提供高保真的渲染环境，更承担了关键的传感器仿真职责。具体来说，主要承担以下功能：

1. **场景构建与更新**
    - **地形渲染**：读取 Simulink 提供的地形设置数据（如高度图和网格参数），动态加载与更新海岸线、海床等三维地形。
    - **大尺度海况**：结合风速、显著波高等宏观参数，使用流场模拟插件渲染真实感极强的海面波浪。
    - **局部特效**：在船体周围自动布置采样点，获取船体局部位置的具体海况信息。
2. **动态对象同步**
    - **船体同步**：持续接收主仿真模块的六自由度位姿指令，实时更新船体的位置和姿态，确保物理运动与可视渲染严格一致。
    - **交通目标**：按脚本或算法输入驱动“来船”模型的运动，为协同测试或碰撞场景提供动态干扰。
3. **气氛与光照**
    - 依据光照高度、太阳方位、云量及天气类型（雨、雾等），自动调节全局光照、阴影、天空盒和环境雾效，为图像感知算法提供真实感强的视觉条件。
4. **传感器级仿真**
    - 集成多种传感器插件（激光雷达、立体／鱼眼相机、红外相机、IMU），在完整渲染场景中进行采样，实时生成点云、图像帧和惯性测量数据。
    - 获取来自Simulink定义的各类传感器参数（视场、分辨率、采样频率、噪声模型）

## 数字孪生
由于**虚拟测试是可重复、可穷举和多批次的测试，具有低成本和高效率的优点**，因此测试场需要配置虚实融合的船舶仿真测试系统。由此，研究院需要开展虚实融合的船舶仿真测试系统研发，除满足自用外，还可推广到无人船海上测试场及其他相关单位。



![岸端模型和船端模型之间进行数字孪生时的逻辑框图](https://cdn.nlark.com/yuque/0/2025/png/2408029/1755508425518-f5ac320b-0954-4a6c-bf54-99071459b7b2.png)



上图是“岸端—船端”协同的数字孪生训练与部署框架。右侧粉色区域是岸端模型：包含数据库、仿真环境与算力中心（内部运行神经网络等算法）；左侧蓝色框是船端模型。中间绿色虚线区域标注为“迭代循环”，表示在岸端完成的试验—反馈—优化—再试验闭环；当闭环达到指标阈值后，再通过专网，将新的神经网络模型，或者是其他算法的参数下发到船端。

船端在真实海况中产生的输入数据与响应数据（传感器观测、控制指令、状态量）先进入数据库。数据库向两条支路供给数据——其一是送入算力中心用于训练与调参与评估；其二是将环境数据（如浪高/周期、风向风速、流速剖面、能见度等）回灌到仿真环境，用于把仿真场景调得更贴近实景。

在岸端模型中，算力中心与仿真环境形成闭环回路：新训练出的神经网络或控制器先在仿真中试验，得到性能与安全性的量化反馈（航迹误差、能耗、极值舵角/推力利用率等），据此继续优化参数或数据集。在部署到船端模型以后，船端运行新的策略并产生新一轮真实数据，回流数据库，闭环继续自我强化。

## 端边协同决策


![岸端模型配合船端模型进行辅助决策的逻辑框图](https://cdn.nlark.com/yuque/0/2025/png/2408029/1755508433317-efde71e6-513c-4eed-abf9-d894060ea42a.png)



这张图展示了一个“岸端—船端”端边协同决策框架。**岸端模型**由“算力中心”和“数据库”组成。船端在一些复杂的情况下，需要请求岸端进行场景/模式判定，在这一过程中，船端首先把**环境数据**（风浪流、能见度、交通密度等）与**状态数据**（位姿、速度、传感器健康、执行器负载等）上传至数据库，同时发起“请分析并配参”的请求；岸端利用算力中心强大的算力，调取历史数据、实时数据和专家经验，利用神经网络完成对模式的识别与决策，再将一整套算法参数组合回传到船端执行。（例如“夜间+视野受阻+强侧风”“港内靠泊+弱流”“GNSS劣化”等），并查表或在线优化出对应的**参数组合**（感知、融合、规划、控制以及可能的神经网络权重/阈值）。这些参数经校验后下发回船端，替换或叠加当前配置。通过这种划分，船端保持“轻量稳态执行”，岸端负责“重计算与全局洞见”。一方面可在多船/多场景间共享经验与数据沉淀，另一方面也使参数更新具有可追溯、可回滚和可审核的工程治理能力。

## 自主避障
本避障分系统旨在实时监测本船航行环境，及时发现潜在碰撞风险，并通过智能决策，引导船舶采取避碰措施，保障航行安全。系统核心功能包括对周边目标船只的动态信息进行持续跟踪与分析，基于航行状态数据（如位置、速度、航向）及预设规则，精确评估碰撞风险。避碰系统采用利用了动态窗口法与人工势能场算法的避碰路径规划方法，综合考虑本船与目标船的动态、以及航行任务，自主生成一条安全的局部避让路径，并可能伴随速度的调整（见图12）。此路径将引导船舶安全绕开障碍。在被动避碰模式下，系统主要依赖于报警模块的风险评估结果。当接收到高级别报警时，系统将采取更为谨慎的减速策略，指令船舶降低航速，甚至在3级报警时执行紧急停车，以此增加避让反应时间或直接消除碰撞动能，同时大致保持原定航线。

抗扰动编队控制分系统旨在确保多艘无人艇在复杂海洋环境（如风、浪、流扰动）下保持稳定编队队形（如直线、V 形、梯队等），同时满足航行安全性与任务执行效率。

层级通信与任务分配分系统旨在构建高效的“岸基-艇群-单艇”层级通信架构，基于任务需求和无人艇能力匹配原则实现任务的合理分配与指令的精准传递，支撑多无人艇协同执行复杂任务（如区域巡逻、目标搜索、物资运输）。
