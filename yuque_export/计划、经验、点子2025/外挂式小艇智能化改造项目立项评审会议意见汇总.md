## 一、项目定位与目标问题
### 1. 项目整体定位不清晰
+ **问题:** 整体听下来很"混沌",逻辑关系没有理清楚,说了很多事情但相互关系不明确
+ **要求:** 需要明确项目的大定位,北鲲02作为CCS认证的重要平台,短期目标是辅助驾驶,中长期目标是开阔水域加全航程

### 2. 外挂式与集成式方案混淆
+ **问题:** 把北鲲二号的集成式改造合到外挂式方案里,导致方案混乱不清
+ **建议:** 考虑是否分开立项,分开写会更清晰,便于管理
+ **核心差异:**
    - 外挂式:追求性价比、快速性、无损改造、可在码头完成
    - 集成式:从内部走穿仓线,与驾控台和机舱集成

### 3. 北鲲02改造目的需明确
+ **要求:** 北鲲02要作为2026年CCS认证的重要平台
+ **配置要求:** 硬件配置可以稍高,接口要多留(如不同类型雷达接口)
+ **适用性考虑:** 要适用定距桨、调距桨等常见推进方式,增强系统生命力

---

## 二、技术方案与产品形态问题
### 4. 产品化程度不足
+ **核心问题:** 倾技术重样机,缺乏工程化条件
+ **具体表现:**
    - 环境适应性未考虑
    - 电源方案未充分考虑
    - 硬件线路像"蜘蛛网",手工焊接水平低
    - 缺乏产品化的机械结构设计

### 5. 外挂式方案核心要素不清
+ **问题:** 最小组成系统和扩展版本界定不清
+ **要求明确:**
    - 基础版:核心控制设备,满足手动遥控切换、跟踪、避障
    - 进阶版:增加感知能力(摄像头、雷达等)
    - 核心部件:谁都要用,谁也跑不掉的部分要明确

### 6. 硬件配置过配问题
+ **问题:** 成本价过高,硬件配置过高
+ **改进方向:**
    - RK3588往下降档次
    - 工控机往下降层次
    - 实现功能不降,成本降低
    - 砍掉不必要的功能(参考马斯克迭代思路)

### 7. 供电方案不足
+ **问题:** 供电是航行系统的底层逻辑,但方案过于简单
+ **需补充:**
    - 能适用什么样的船
    - 系统功率密度
    - 船上电子设备的适配性
    - 方案需要简化

---

## 三、技术指标与性能要求
### 8. 技术指标不合理
+ **具体问题:**
    - "12小时装上,30分钟拆掉"不现实(实际装了两个月)
    - 指标缺乏责任心,不切实际

### 9. 性能指标要求
+ **槽道舟:** 追求性价比的指标
+ **北鲲02:**
    - 如果作为取证平台,指标不能太低
    - 需要咨询相关专家(底工)明确CCS认证要求

### 10. 感知设备配置
+ **雷达接入问题:**
    - 日本古野雷达等高端设备难以接入
    - 建议用盒子接出ARPA信息和海图数据
+ **摄像头配置:**
    - 基础版摄像头数量较少
    - 集成式增加前视拼接(≥220度,3个相机)和360度环视(4个鱼眼)

---

## 四、软件架构与规范问题
### 11. 软件架构不明确
+ **问题:** 没有统一的软件架构
+ **要求:**
    - 明确硬件架构和软件架构
    - 确定主要软件的运行环境和编写语言
    - 统一编程规范(建议C + Python两种语言)
    - 航控所要作为"一盘棋"统一考虑

### 12. 代码规范缺失
+ **问题:** 之前开发过于自由,缺乏规范
+ **改进:**
    - 统一软件编程规范
    - 便于移植和维护
    - 从demo阶段转向可复制的规范化产品

### 13. 产品稳定性和可维护性
+ **问题:**
    - 遗传船经常出现故障(显示屏坏、控制失灵、主机停机等)
    - 需要长期驻场售后支持
+ **要求:**
    - 产品稳定化,减少现场驻场人员
    - 支持远程操作和在线升级
    - 远程监控和故障诊断功能

## 五、文档质量问题
### 14. 命名和术语问题
+ **"外挂式"命名:** 过于口语化,不够专业
+ **技术指标表述:** 船只参数错误(5米双水星3人3节,尺寸长度都不对)
+ **文档质量:** 属于"拼凑",内容不严谨

### 15. 案例和数据准确性
+ **5米船案例:** 参数错误,查到的数据是10米
+ **文档来源:** 部分内容从其他文档抄来,未核实

## 六、预算与投资问题
### 16. 预算概念错误
+ **问题:** 只列了硬件成本,没有预算投资概念
+ **预算投资 vs 硬件成本:** 两者差距巨大
+ **费用构成不全:**
    - 外挂基础版:约7.8-7.9万(仅硬件)
    - 外挂进阶版:约13.7万(仅硬件)
    - 北鲲02集成式:约17.6万(仅硬件)
+ **缺失内容:** 人工成本、测试费用、认证费用等

## 十、评审总结要求
### 23. 下次评审需要明确的5个核心要素
1. **产品形态:** 明确最终产品是什么,随时可以销售的标准
2. **产品架构:** 软硬件架构如何设计
3. **CCS认证:** 如何考虑认证要求
4. **实验平台定位:** 北鲲02作为实验/升级平台的定位
5. **实验方案:** 如何进行测试和验证

### 24. 产品定义标准
**产品必须满足:**

+ 随时可以销售
+ 实验充分
+ 性能验证完成
+ 供应链明确

### 25. 整体评审态度
+ **不是否定立项,而是要求说清楚**
+ **方向走偏代价很大**
+ **这是2026年能否拿到CCS认证的关键项目之一**
+ **要慎之又慎**
